# ANN Implementation Plan

## Goal

Turn the `ann` domain into a real, reproducible parameter-generation flow for the `4 -> 8 -> 1` MLP with one trained source of truth for downstream domains.

## Scope

- Dataset generation is deterministic and integer-valued.
- Inputs are sampled from `[-16, 15]`.
- Labels are generated by a fixed ReLU teacher network, which is a concrete piecewise-linear rule.
- The default training command writes one selected run to `ann/results/latest/`.
- The ANN domain does not generate downstream artifacts (RTL ROM, Lean spec, test vectors). That responsibility belongs to the `contract` domain.
- The unified CLI entrypoint lives at `ann/cli/__main__.py`.

## Execution Plan

1. Generate a deterministic dataset snapshot for train / validation splits.
2. Train the `4 -> 8 -> 1` ReLU MLP with deterministic mini-batch Adam updates.
3. Evaluate floating-point and quantized checkpoints on train / validation data.
4. Select one downstream checkpoint from quantized validation metrics.
5. Write selected artifacts to `ann/results/latest/`.

The CLI may optionally invoke the `contract` domain's freeze step after training, but the ANN domain itself only writes to `ann/results/`.

## Default Configuration

- Seed: `20260312`
- Optimizer: Adam
- Batch size: `64`
- Epoch budget: `300`
- Early stopping patience: `20`
- Initialization: deterministic random uniform

## Selection Policy

The selected downstream checkpoint is chosen by:

1. Higher quantized validation accuracy
2. Lower quantized validation loss
3. Smaller quantized `L1` magnitude

This keeps the chosen result tied to deployable integer inference rather than only floating-point quality.

## Output Artifacts

The default training flow produces:

- `ann/results/latest/weights_float.json`
- `ann/results/latest/weights_float_selected.json`
- `ann/results/latest/weights_quantized.json`
- `ann/results/latest/metrics.json`
- `ann/results/latest/training_summary.md`
- `ann/results/latest/dataset_snapshot.jsonl`

## Current Implementation Status

- The repository currently uses a pure-Python implementation of Adam and mini-batch training so the flow runs without extra dependencies.
- The training loop uses quantization-aware forward passes with a straight-through style gradient approximation.
- The default inference path loads the canonical contract weights from `contract/result/weights.json` (a file produced by the `contract` domain).
- The unified CLI exposes `train`, `evaluate`, and `quantize` via `python3 -m ann.cli`. The `export` subcommand delegates to the `contract` domain's freeze step.
